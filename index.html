NETWORK PROTOCOL DESIGNER

I proposed a topology discovery protocol for initializing a network deployable autonomous underwater system in which the nodes have no knowledge of the system topology upon deployment. The goal is to establish multi-hop, minimum-power acoustic communication links over a given coverage area. The protocol base on the uniqueness of node IDs, adopt TDMA mechanism to remove the risk of collision of topology discovery packets and then reduce the energy consumption.

THE 12th FIVE-YEAR PRE-RESEARCH PROJECT NOVEMBER 2016 - MAY 2017

TIME SYNCHRONIZATION AND NODES LOCATION

Research on network initialization.The initialization method is an important procedure for network protocol. It has two parts, self-localization and time synchronization. I had researched a self-localization algorithm with selected reference nodes only using on e anchor. This algorithm reduces the influence of topology and range error. It has a high accuracy without using a lot of anchors.
Besides, Unlike earlier range-based 3D localization techniques that required distance estimates from at least four reference nodes to localize any node, LOTUS can approximate locations with two references. This enables LOTUS to work in low density deployment cases and allows more nodes to be localized.
In regard to time synchronization, we solve the time synchronization and localization problems at the same time, which can decrease the time delay of the initialization process and the communication traffic.
